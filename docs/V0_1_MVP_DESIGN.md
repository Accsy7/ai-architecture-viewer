# AI Architecture Viewer v0.1.0 历史设计说明

> 本页仅保留早期产品探索的历史背景，不是当前操作指南。当前产品与协议以 [README](../README.md)、[v0.6 兼容说明](pending-change-layer-migration.md) 和 [CHANGELOG](../CHANGELOG.md) 为准。

v0.1.0 曾探索过“查看器内嵌模型、选择资料、生成候选提案、逐项人工处理”的方向。该方向从未成为当前架构，并已先后被 v0.2 的“外部编码智能体 + MCP”定位和 v0.6 的 direct-draft 流程取代。

当前不可混淆的边界如下：

- 查看器不内嵌模型、不要求模型 API Key，也不替外部智能体扫描仓库。
- Codex、Claude Code 等外部智能体在自己的对话中与用户讨论，并提交有依据的稳定 ID 语义补丁。
- 服务端只把验证通过的补丁写入运行锁定的活动草稿；过期或冲突写入会被拒绝。
- 当前架构草稿只能由 `code-fact` 支持；讨论、设计文档与明确标注的推断只能支持目标设计。
- 草稿写入不是人工批准，也不产生新的逐项审批队列。
- 用户在画布检查完整草稿及其相对正式版的净变化后，执行一次明确发布；发布是唯一的人工架构门槛。
- 外部智能体只能读取已发布正式目标作为开发合同，不能发布架构或验收自己的实施结果。
- v0.2–v0.5 的旧提案与决定继续作为只读兼容历史保存，不会自动迁移成草稿或正式版本。

早期设计中的内嵌模型、资料扫描、提案收件箱和逐项审批 API 均已退役，不应被新客户端、Skill 或界面重新接入。
