# v0.6.1 visual convergence and legacy-group compatibility

This release is a presentation and compatibility update on top of the accepted v0.6 direct-draft baseline. It does not change the draft, publication, formal-contract, implementation-review, or MCP governance model.

## Metadata compatibility

- `meta.groups` remains the canonical and preferred region definition.
- When `meta.groups` is absent and `meta.capabilityDomains` is an array, the read/migration layer exposes a cloned `meta.groups` array with every legacy field preserved.
- An explicitly present `meta.groups`, including an empty array, is never replaced by `capabilityDomains`.
- The original `capabilityDomains` field remains available for traceability; migration does not delete or rewrite it.
- Region membership continues to use the stable `node.data.group` value. Coordinates never infer membership.
- Region geometry continues to prefer saved layout containers, then group metadata defaults. Region nodes render behind architecture cards and retain drag/resize behavior.

## Draft-change presentation

- The canvas header shows one compact unpublished-draft sentence and only the actions to inspect changes or review publication.
- Counts, provenance explanations, fields, and evidence move to a temporary right-side inspector that replaces module details while open.
- The inspector groups modules, relationships/boundaries, and acceptance criteria. Group and item rows remain collapsed until the user requests detail.
- Publication preview remains the complete final review surface.
- Draft decorations do not replace a card's normal background or border. Add/change states use a narrow marker and a small badge; removal may reduce opacity while preserving readability.

## Desktop sizing and language

- The non-fullscreen desktop workspace is exactly `880px` high. Canvas and inspector share that height; the page itself may scroll normally.
- Fullscreen remains viewport-bound and overrides the fixed desktop height.
- The language control contains one text button: Chinese mode displays `English`, and English mode displays `中文`.
- Language persistence and diagram/view identity are unchanged. Project-owned content is never translated.

## Non-regression boundaries

- Agent writes remain stable-ID semantic patches against an exact published/draft lock.
- Drafts never appear in `get_approved_target` and never become implementation contracts before local publication.
- Agents cannot write layout/routing fields, publish architecture, or review implementation.
- MCP remains the same nine-tool governed surface.
- v0.2-v0.6 state, analysis, proposal, review, and revision history remain readable.
